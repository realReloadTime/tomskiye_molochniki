Регистрация и вход в аккаунт это две разных ссылки. То есть, пользователь после того как зарегистрировался не войдет сразу в свой личный кабинет, ему нужно будет заходить в аккаунт.


post/api/Registration
Для регистрации должно мне приходить:  
{
  "login": string,       
  "password": string,    
  "confirmPassword": string 
}

post/api/Auth/login
Для входа должно мне прихлдить:  
{
  "login": string,       
  "password": string   
}

Для выхода из аккаунта ссылка:
post/api/Auth/logout

Для проверки авторизации:
GET /api/Auth/check
Вы от него получаете:
{
  "isAuthenticated": boolean, <-- Тру, Фолз Авторизован или нет;
  "userId": number,          
  "login": string             
}
Вы его вызываете при загрузке приложения
Если авторизован, то сразу открываете личный кабинет. Если не авторизован, то обычную страницу.
В личном кабинете так же должна быть функция для загрузки файлов.

Кукис отправляются автоматически!!!
И кукис и сессия длится 30 дней

Валидации полей на бэке нет! Проверку длины пароля, символов в логине и т.д. Должно быть на фронте. 
От меня вы получаете статусные коды со строко:

BadRequest("Пользователь с таким логином уже существует");  post/api/Registration
BadRequest("Пароли не совпадают");  post/api/Registration
Ok("Пользователя зарегистрирован");  post/api/Registration
Ok("Выход выполнен успешно"); post/api/Auth/logout
return Ok("Успешный вход");  post/api/Auth/login
